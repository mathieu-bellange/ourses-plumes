Conclusion

Surprisingly, various benchmarks on major browsers reveal that :

  1. Base-64 embeded web-font is slower than external linking with @font-face {src: url('font.ext');} for HTML texts.
  2. Base-64 embeded web-font is slightly slower than external linking with or @import url("file.css"); for SVG texts, but the difference is so tiny that it does not worth the loss of interoperability (despise the extra weight of code for font in each SVG file).

No smart solution is to be found. Web fonts remain in any case a real heavy load for both client and server.

Nonetheless, a least than worst workaround for web fonts is as mentioned above :

  • External web-font files for HTML texts.
  • Base-64 embeded web-font for SVG texts.
